# ひかりなぞり

カメラに**手をかざすと、指先や手のひらから光があふれる**体験型ミニアプリ。手のジェスチャーで光をあやつります。

> 手をかざすと、光があふれる。ひらく・つまむ・にぎる で あやつる。

素の HTML / CSS / JavaScript（Canvas 2D）＋ 手の認識に [MediaPipe Hands](https://developers.google.com/mediapipe) を使用。ビルド不要。

---

## 体験の核

- カメラに手を写すと、**伸ばした指先から光の粒が流れ出す**（手を動かすと光がついてくる）
- **手をひらく**と、手のひらから光がブワッと広がって咲く
- **つまむ**（親指＋人差し指）と光が集まり、**離す**とぱっと弾ける
- **にぎる**（グー）と、光がふわっと鎮まる
- 光は生きているように流れ、ゆっくり咲いてから消える（手を止めても動き続ける）

手が認識できない時・カメラが無い時は、**指タッチ（PCはマウス）でそのまま描けます**。

---

## 起動方法

ビルド不要。`light-veil/index.html` を開くだけで動きます。
ただし**カメラは HTTPS（または localhost）が必要**、**手の認識モデルはネット接続が必要**（CDNから読み込み）です。ローカルでは簡易サーバー経由がおすすめ。

```bash
# このフォルダ（light-veil）の中で
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

`localhost` はブラウザがセキュアな扱いをするので、ローカルでもカメラが動きます。

---

## PC での確認方法

- 上記サーバーで `http://localhost:8000` を開く（カメラOK）
- PC内蔵カメラに手をかざせばジェスチャーが動きます
- 手が無くても、**マウスのドラッグで光を描く／クリックで弾ける**で確認できます

---

## スマホでの確認方法

### 1. 同じ Wi-Fi のローカルネットワーク

```bash
python3 -m http.server 8000 --bind 0.0.0.0
```

PC の IP（例 `192.168.0.12`）を調べ、スマホで `http://192.168.0.12:8000/light-veil/` を開きます。
⚠️ これは **http** なので、スマホではカメラが動かないことがあります（下記）。その場合は HTTPS 公開で確認してください。

### 2. QR コード

確認用 URL を QR 化すると開きやすいです。

```bash
qrencode -t ANSIUTF8 "https://<あなたの公開URL>/light-veil/"
```

### 3. HTTPS で公開（カメラ＆手の認識を試すならこれが確実）

GitHub Pages / Netlify / Vercel に置けば HTTPS になり、スマホでカメラ＆手の認識が動きます（手順は下記）。

## カメラ権限について

- カメラは**ユーザーが「カメラを使う」を押したあと**に初めて要求します（勝手に起動しません）。
- 許可しなくても、**写真モード**＋指タッチで遊べます。
- 手をかざす体験には**内カメラ（自撮り）**が自然なので、最初は内カメラで起動します。右上のカメラアイコンで外カメラにも切り替えられます（内カメラは鏡像表示）。

---

## HTTPS が必要な理由

スマホのブラウザは、セキュリティ上 **`getUserMedia`（カメラ取得）を「セキュアな文脈」でしか許可しません**。

- ✅ `https://...`（公開サイト）
- ✅ `http://localhost`（開発用に特別扱い）
- ❌ `http://192.168.x.x`（ただの http はカメラ不可のことが多い）

スマホでカメラ＆手の認識を確認するときは **HTTPS 環境**を使ってください。ローカル http の場合は写真モード＋指タッチでどうぞ。

### 手の認識モデルについて

手の認識は MediaPipe のモデルを **CDN（jsdelivr / Google）から読み込みます**。
- 初回だけ数MBのダウンロードがあります（ネット接続が必要）
- 読み込めない環境では自動で**指タッチ描画にフォールバック**します
- 最近のスマホ推奨（GPUを使います）

---

## 写真モードの使い方

1. 下部の「写真」を選ぶ（または最初の案内で「写真を選ぶ」）
2. 端末の写真を選択
3. 写真の上を**なぞる／タップ**して光を描く（写真モードは指タッチ操作）
4. 右上の**保存ボタン**で、光ごと1枚の PNG にして保存（難しい端末はスクショでもOK）

写真モードは PC・スマホどちらでも動きます（HTTPS 不要）。

## 操作一覧

### 手のジェスチャー（カメラ＋手の認識オン時）
- **指を伸ばして動かす**：指先から光の粒が流れる
- **手をひらく**：手のひらから光が広がって咲く（手前と奥へ立体的に散る）
- **つまむ→離す**：光が集まってから、手前へぱっと弾ける
- **にぎる（グー）**：光がふわっと鎮まる
- **指さし（人差し指だけ）**：集中した一筋の光（光のペン）
- **サムズアップ（親指だけ）**：金色の星がパッと舞う
- **手を近づける／引く**：光が手前へ飛び出す／奥へ吸い込まれる（遠近感）
- **両手をかざす**：手のひらの間に光の橋が架かる

### 遠近感について
光は深度を持ち、画面中心を消失点にした遠近投影で描かれます。**手前に来る光は大きく明るく、長めに残留**し、**奥へ行く光は小さく淡く、すっと消えて**いきます。手を近づける／引く、開く、つまんで離す、で奥行き方向に光が飛びます。

### 指タッチ／マウス（いつでも）
- **なぞる**：軌跡に沿って光が流れる
- **タップ／クリック**：その場で光が弾ける

### ボタン
- **音（左上）**：やさしい音のオン／オフ。指先＝風鈴、ひらく＝和音、つまむ＝集める音、離す＝弾ける音、にぎる＝鎮まる音
- **カメラ切替（右上）**：内カメラ（自撮り・鏡像）⇄ 外カメラ
- **手で あやつる（右上の手アイコン）**：手の認識のオン／オフ
- **星座づくり（右上の星アイコン）**：下記の星座モードのオン／オフ
- **消す（右上）**：光を消す
- **保存（右上）**：今の見た目を1枚の画像として保存

### 星座づくり（ゆるい創作モード・失敗なし）
オンにすると、景色の中にかすかな星（点）がいくつか浮かびます。**指先（手の認識オン時）やタッチで星に触れると、ひとつずつ灯って線でつながり**ます。
- つなぐ順番は自由。触れた順に線が伸び、星をともすたびに**音階が一段ずつ上がって旋律が育ちます**
- すべての星をつなぐと**「星座 かんせい！」**＝和音が鳴り、星座がふわっと昇って消え、次の星座が現れます
- 制限時間・スコア・失敗はありません。光遊びと同時に楽しめます

### 色テーマ（下部）— 見た目と「音」が変わります
- **みどり** … 緑〜シアンの発光／澄んだ鈴と厚みのある和音
- **オーロラ** … 緑〜青〜紫がゆらめく／残響の深いエアリーな響き
- **ゴールド** … あたたかい金色の光／低音の効いたベルの響き

音は単純な電子音ではなく、**デチューンを重ねた発振＋長いリバーブ＋低音＋ゆっくり揺れるアンビエントの和音ドローン**で、壮大で上品な響きにしています（最初のタッチで静かに立ち上がります）。エフェクトも白飛びを抑え、繊細なブルームと控えめなきらめきで上品にまとめています。

## GitHub Pages で公開する場合

このリポジトリは静的サイトなので、そのまま公開できます。

1. リポジトリの **Settings → Pages**
2. **Source** を `Deploy from a branch` にして、公開したいブランチ（例 `main`）と `/ (root)` を選択
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/light-veil/` で公開されます

Netlify / Vercel もビルドコマンドなしの静的サイトとしてデプロイすればOK（HTTPS が自動で付くのでカメラ＆手の認識が動きます）。

## 子ども向け・安全性について

子どもに見せても安心な表現にしています。

- 暴力・怖い表現なし／激しい点滅なし（光はやわらかくフェード）
- 音はやわらかい音色のみ（耳ざわりな音・急に大きくなる音なし）。最初のタッチのあとに鳴り、左上でオフにできます
- **顔認識・個人特定はしません**。手の認識は「手の形（指先・関節の位置）」だけを見るもので、本人を識別しません
- **映像も手のデータも、保存・外部送信しません**。画像処理はすべてブラウザ内で完結します
- カメラ権限はユーザー操作のあとにだけ要求し、拒否しても写真モード＋指タッチで遊べます

## 技術メモ

- Canvas 2D ＋ MediaPipe Hands（手のランドマーク21点）。光はバニラ JS
- リッチな見た目を軽量に：光は専用バッファに加算→**1/4解像度ブルーム**で柔らかく発光、**star flare のきらめき**、速い粒の**モーションストリーク**、**周辺減光（ビネット）**で光を際立たせる。重いシェーダーは不使用
- 光は色相ごとに事前生成した発光スプライト＆きらめきスプライトを `drawImage` で重ねるだけ＝軽い（毎フレームのグラデ生成なし）。実機相当で激しい操作中も約60fps
- 粒はゆるい渦（curl風の流れ）で動き続け、加算合成で発光。数秒でフェード
- 背景（カメラ/写真）と光を1枚のキャンバスに合成 → そのまま保存できる
- 手のランドマークは cover フィット＋鏡像を反映して画面座標へ変換
- `touch-action: none` ＋ `touchmove` 抑制でスクロール暴発を防止。`devicePixelRatio` は 1.5 まで、粒は上限つき
