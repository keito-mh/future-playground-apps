# ひかりなぞり

カメラ映像や写真の上を指でなぞると、緑〜シアンの**光の粒・泡・半透明の膜**がふわっと広がる、触って気持ちいい体験型ミニアプリ。

> なぞると、光の膜がひろがる。タップで咲く、長押しで溜める。

素の HTML / CSS / JavaScript（Canvas 2D）だけで動きます。ビルド不要。

---

## 起動方法

ビルドは不要です。`light-veil/index.html` を開くだけで動きます。

ただし**カメラ機能を使うには HTTPS（または localhost）が必要**なので、ローカルでは簡易サーバー経由がおすすめです。

```bash
# このフォルダ（light-veil）の中で
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

`localhost` はブラウザがセキュアな扱いをするので、ローカルでもカメラが動きます。

---

## PC での確認方法

- 上記のサーバーを立てて `http://localhost:8000` を開く（カメラOK）
- **マウスでも操作できます**
  - ドラッグ＝なぞる / クリック＝タップ / 押しっぱなし＝長押し
- 写真モードは PC でもそのまま遊べます

---

## スマホでの確認方法

### 1. 同じ Wi-Fi のローカルネットワークで開く

PC で次のように起動すると、スマホからアクセスできます。

```bash
# ローカルIPでサーバーを公開
python3 -m http.server 8000 --bind 0.0.0.0
```

PC の IP アドレス（例 `192.168.0.12`）を調べて、スマホのブラウザで
`http://192.168.0.12:8000/light-veil/` を開きます。

> ⚠️ この URL は **http** なので、スマホでは**カメラが動かないことがあります**（下の「HTTPSが必要な理由」参照）。
> その場合は**写真モード**で遊ぶか、HTTPS で公開して確認してください。

### 2. QR コードで開く

確認用 URL を QR 化するとスマホで開きやすいです。

```bash
# 例：qrencode を使う場合
qrencode -t ANSIUTF8 "https://<あなたの公開URL>/light-veil/"
```

オンラインの QR 生成サービスに URL を貼り付けてもOKです。

### 3. HTTPS で公開して確認（カメラを使うならこれが確実）

GitHub Pages / Netlify / Vercel などに置けば HTTPS になり、スマホでもカメラが動きます（下記手順）。

---

## カメラ権限について

- カメラは**ユーザーが「カメラを使う」を押したあと**に初めて要求します（勝手には起動しません）。
- 許可しなくても、**写真モード**でそのまま遊べます。
- 背面カメラ（`environment`）を優先して使います。

---

## HTTPS が必要な理由

スマホのブラウザは、セキュリティ上の理由から **`getUserMedia`（カメラ取得）を「セキュアな文脈」でしか許可しません**。

- ✅ `https://...`（公開サイト）
- ✅ `http://localhost`（開発用に特別扱い）
- ❌ `http://192.168.x.x`（ただの http はカメラ不可のことが多い）

そのため、**スマホでカメラを確認したいときは GitHub Pages / Netlify / Vercel などの HTTPS 環境**を使ってください。ローカル http で確認する場合は写真モードをどうぞ。

---

## 写真モードの使い方

1. 下部の「写真」を選ぶ（または最初の案内で「写真を選ぶ」）
2. 端末の写真を選択
3. 写真の上を**なぞる / タップ / 長押し**して光を描く
4. 右上の**保存ボタン**で、エフェクトごと1枚の PNG にして保存
   - うまく保存できない端末では、そのまま**スクリーンショット**でも残せます

写真モードは PC・スマホどちらでも動きます（HTTPS 不要）。

---

## 実装したモード一覧

下部のタブで質感を切り替えられます。

エフェクトの主役は「写っている景色そのものの歪み」です。なぞった場所の背景が、水面のようにふくらんだり・流れたり・波紋になったりします。

| モード | 内容 |
| --- | --- |
| **ベール** | なぞった景色がやわらかくふくらむ、膜のような歪みモード |
| **フロー** | 指の動きに沿って、景色がすーっと流れて歪むモード |
| **レンズ** | タップ・なぞりから波紋が広がり、水面のように揺れるモード |

操作はどのモードも共通です。

- **なぞる**：軌跡に沿って光の粒と膜が広がる（速いほど粒が多く帯が太い／ゆっくりだと遅れて集まる）
- **タップ**：その場に光の泡がぱっと咲き、波紋が広がる
- **長押し→離す**：指の周りに粒が溜まり、離すとふわっと散る
- **リセット（右上）**：画面の光を消す
- **保存（右上）**：現在の見た目を1枚の画像として保存
- **音（左上）**：やさしい音のオン／オフ。なぞると風鈴のような粒の音、タップで水滴、長押し→離すと和音でふわっと開く

歪みの境界はパキッと切れず、外側へ向けてグラデーションでなめらかに溶けるようにしています。

---

## GitHub Pages で公開する場合

このリポジトリは静的サイトなので、そのまま公開できます。

1. リポジトリの **Settings → Pages**
2. **Source** を `Deploy from a branch` にして、公開したいブランチ（例 `main`）と `/ (root)` を選択
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/light-veil/` で公開されます

Netlify / Vercel の場合も、ビルドコマンドなしの静的サイトとしてフォルダごとデプロイすればOKです（HTTPS が自動で付くのでカメラが動きます）。

## 子ども向け・安全性について

子どもに見せても安心な表現にしています。

- 暴力・怖い表現なし／激しい点滅なし（光はやわらかくフェード）
- 顔認識・個人識別は一切しません
- **写真やカメラ映像を外部に送信しません**。画像処理はすべてブラウザ内で完結します
- カメラ権限はユーザー操作のあとにだけ要求し、拒否しても写真モードで遊べます
- 顔を変形させるような強い加工はしません（歪みはごく弱く、境界もなめらかに溶けます）
- 音はやわらかい音色のみ（耳ざわりな音・急に大きくなる音なし）。最初のタッチのあとに鳴り、左上のボタンでいつでもオフにできます

## 技術メモ

- Canvas 2D、ライブラリ非依存（バニラ JS）。軽さ最優先の設計
- エフェクトの主役は背景の歪み：円形にクリップして景色を拡大／ずらして描き直すだけ（GPU の `drawImage`）。重いピクセル処理なし
- 光は最初に一度だけ作った「光スプライト」を `drawImage` で置くだけ。毎フレームの `createRadialGradient` を排除
- 全画面の残光バッファも廃止。各エフェクトは `life` で自然に消える
- 写真モードで何も触っていない時は描画自体をスキップ（電池にやさしい）
- 背景（カメラ/写真）と歪み・光を1枚のキャンバスに合成 → そのまま保存できる
- `touch-action: none` ＋ `touchmove` 抑制で、なぞり中のスクロール暴発を防止
- `devicePixelRatio` は 1.5 までに制限し、歪み・粒の数にも上限を設けて端末を保護
- iPhone 実機相当（390×844）で、激しいドラッグ中も約 60fps を確認
